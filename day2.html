🌿 Day 2 · 文件操作已完成

学习日期:2026-04-27

🎯 学习目标:用纯键盘新建、读取、追加、删除一个真实的文本文件。理解 > 和 >> 这两个"重定向"符号的威力。

第 1 章

从"文件夹"升级到"文件"

Day 1 我们都在文件夹层面玩(进出文件夹、新建文件夹)。Day 2 升级到文件层面:

  • 怎么新建一个文件?
  • 怎么写字进去?
  • 怎么读它?
  • 怎么删掉它?

而且你会学到一个超酷的符号 >——它能把命令的输出直接写进文件。这是终端的灵魂之一。

第 2 章

cls + echo — 清屏 + 打印

cls 清屏

clear screen。屏幕乱了,输 cls 立刻干净。

echo "内容" 打印一句话

echo = 回声。你说啥它打印啥。

PS C:\xxx> echo "我是大佬"
我是大佬

看起来没啥用对吧?它的真正威力在第 3 章——配合 > 符号。

第 3 章

> 重定向 — 把输出装进文件

命令默认输出到屏幕。但加个 > 符号,可以让输出"流向文件"。

覆盖写入(单 >)

echo "今天我学会了用终端" > diary.txt

这条命令的意思:

  1. 新建一个叫 diary.txt 的文件(如果存在则清空覆盖)
  2. 把"今天我学会了用终端"这段文字写进去

屏幕没有任何反应,但文件已经创建了。可以用 dir 验证。

追加写入(双 >>)

echo "明天我要学 Python" >> diary.txt

>> 是追加——保留原文件内容,新内容加到末尾。

⚠️ 区分:> 和 >>
> = 覆盖(原内容会消失!)
>> = 追加(原内容保留,新的接到后面)
真实文件操作中,> 用错可能让你丢失重要数据。先用 >> 思考一下。
第 4 章

type — 看文件内容

查看文件里写了啥。

PS C:\xxx> type diary.txt
今天我学会了用终端
明天我要学 Python

注意它原样输出到屏幕,什么编辑器都不用开。

💡 类比:type 在 Linux/Mac 上叫 cat。是个不同名字的同一个工具。
第 5 章

del 和 Remove-Item — 删除

del 文件 删文件

del diary.txt

Remove-Item 文件或文件夹 通用删除

PowerShell 风格的命令,文件、文件夹都能删。

Remove-Item my-folder
⚠️ 警告:终端删除是永久删除
用 del 或 Remove-Item 删的文件不会进回收站,删了就找不回来。
对真实文件要谨慎。先 dir 看清楚再删。
第 6 章

真实踩坑记录

坑 1:echo 后面忘了空格
❌ echo"内容">a.txt → 报错 "无法识别为 cmdlet"
✅ echo "内容" > a.txt
原因:终端按空格区分"命令"和"参数"。没空格 → 整个被当成一个命令名 → 找不到 → 报错。
坑 2:type 看 Markdown / 中文文件出现乱码
老版 PowerShell 默认用 GBK 编码读文件,但很多文件保存的是 UTF-8,两边对不上 → 中文显示成 锛佺潃 这种乱码。
解决:① 用 Get-Content -Encoding UTF8 文件名 ② 或者直接用 VSCode 打开 ③ 写 PowerShell 命令时,先 chcp 65001 切换编码

🏆 解锁的能力

  • 能用纯键盘新建、读取、追加、删除一个文本文件
  • 理解 >(覆盖)和 >>(追加)的区别
  • 知道终端删除不进回收站,操作时会更谨慎
  • 认识到"编码"这个概念,知道 GBK 和 UTF-8 是不同的"字典"
  • 知道报错先看最后一两行,关键词通常在 Exception 那行
