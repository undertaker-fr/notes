🚀 特别任务 · 让笔记上线已完成

学习日期:2026-04-27

🎯 学习目标:把"只在自己电脑里的"HTML 文件,部署成"全世界都能访问的"公网网站。学会 Git 和 GitHub 的基础工作流。

第 1 章

为什么别人现在看不到你的网页?

你做好的 index.html 文件只存在你电脑里。别人要看,需要:

  1. 把这个文件上传到一台永远开机、连着公网的电脑(叫服务器)
  2. 那台服务器给你一个网址(比如 https://你的名字.github.io/notes)
  3. 别人在浏览器输这个网址 → 服务器把你的文件发给他们 → 显示

这个过程叫部署(Deploy)。听起来高级,但有现成的免费工具,5 分钟就能搞定。

第 2 章

核心概念

Git 给文件每次改动"拍快照",像时间机器,随时能回到任何历史版本。它装在你电脑里。
GitHub 程序员的"代码版朋友圈",全球 1 亿+开发者在上面存代码。它是网站。
仓库(Repo) 一个项目的容器,既存文件,也存所有历史快照。
部署(Deploy) 把文件放到永远开机的服务器,让别人能通过网址访问。
GitHub Pages GitHub 提供的免费网页托管服务,把公开仓库变成网站。
main 分支 Git 仓库的主线,默认大家把代码推到这里。
💡 关系:Git(本地工具)和 GitHub(在线网站)是不同的东西。Git 是工具,GitHub 是用 Git 的网站。两者经常一起用,但不是同一个东西。
第 3 章

核心命令

命令功能例子
git init -b main把当前文件夹变成 Git 仓库,主分支叫 maingit init -b main
git add 文件把文件加入"待保存清单"git add index.html
git commit -m "..."把待保存清单"拍照存档",信息写在引号里git commit -m "首次提交"
git push把本地存档"推"到 GitHub 云端git push
gh repo create用 GitHub CLI 直接建远程仓库gh repo create notes --public --push

本地 → GitHub 的核心三步

一句话总结:add → commit → push

  1. add:挑选要保存的文件(可以选一个、几个或全部)
  2. commit:给这一批改动拍张"快照",写一句话描述
  3. push:把"快照"推到 GitHub 云端
第 4 章

我们今天具体做了啥

  1. 在 F:\Claude code\notes 建文件夹,把 我的编程笔记.html 复制成 index.html
  2. git init -b main 初始化 Git 仓库
  3. git add index.html + git commit -m "..." 第一次拍快照
  4. gh repo create undertaker-fr/notes --public --push 在 GitHub 建仓库并推送
  5. 用 gh api 启用 GitHub Pages 功能
  6. 等 1-2 分钟构建完成 → 访问 https://undertaker-fr.github.io/notes/
第 5 章

关键认知

  • commit message(提交信息)不是给电脑看的,是给未来的自己和队友看的。要写清楚改了啥
  • Public 仓库 = 全世界都能看代码;Private 仓库 = 只有你和邀请的人能看。GitHub Pages 的免费版要求 Public
  • GitHub Pages 网址规则:https://用户名.github.io/仓库名/
  • 第一次部署 1-3 分钟,后续更新只要几十秒
  • 查构建状态:gh api repos/用户名/仓库名/pages/builds/latest
第 6 章

关于"中国能不能直接打开"

因为 GitHub 的服务器在国外,国内访问 github.io 域名会慢、有时打不开。要让国内朋友秒开,可以叠加一层国内 CDN:

  • Gitee Pages(码云):中国版 GitHub,需实名认证
  • 腾讯云 EdgeOne Pages:免费、快、可以从 GitHub 自动同步

这一步是进阶选项,等你 GitHub 玩熟了再加。
