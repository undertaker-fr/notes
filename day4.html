第 1 章

变量是什么?

一句话

变量 = 一个贴了标签的盒子,里面装着数据,你以后用标签的名字就能拿到数据。

盒子比喻

┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 标签:name    │    │ 标签:age     │    │ 标签:city    │
│             │    │             │    │             │
│  "小明"      │    │   18        │    │  "上海"      │
└─────────────┘    └─────────────┘    └─────────────┘
  • 盒子上的标签= 变量名(name、age、city)
  • 盒子里装的东西= 变量的值("小明"、18、"上海")

为啥需要变量?

不用变量,改一个名字要改很多地方:

print("欢迎,小明!")
print("小明,你今年 18 岁。")
print("小明住在上海。")

想换"小明"为"张三",得改 3 次。100 行代码就要改 100 次。

用变量,只改第一行,后面自动跟着变:

name = "小明"
print("欢迎," + name + "!")
print(name + ",你今年 18 岁。")
print(name + "住在上海。")

"改一处,处处生效"——这就是变量的威力。

变量的 3 个核心特点

  1. 有名字也有值:名字是标签,值是盒子里的内容
  2. 可以反复使用:就像贴了"袜子"标签的抽屉,每次找袜子都看那个标签
  3. 值可以改:这就是叫"变量(variable)"的原因——它的值是可变的

变量 vs 字符串(关键区分)

print("hello")    ← "hello" 是字符串(直接的文字)
print(hello)      ← hello 没引号 → Python 当成变量名找
  • 有引号→ 字符串(就是字面这段文字)
  • 没引号→ 变量名(去找有这个名字的盒子)
🤔 思考题:下面代码会输出什么? 
price = 100
print(price)
print("price")
看参考答案
两行:
第 1 行 100(变量,从盒子取值)
第 2 行 price(字符串,原样输出 5 个字母)
第 2 章

怎么创建变量?= 的真正含义

基本语法

变量名 = 值

读作:"把右边的值,放进左边的盒子"。

⚠️ 重要警告:Python 的 = 不是数学的 =

数学课上 = 表示"两边相等"。Python 的 = 是"赋值"——一个动作:

name = "小明"
       ↑      ↑
   把右边的"小明"放进左边这个叫 name 的盒子
数学Python
x = 5"x 等于 5"(陈述)"把 5 放进 x 盒子"(动作)
x = x + 1❌ 不可能✅ "取 x 当前值加 1,再放回 x"

专业名词:= 在 Python 里叫赋值运算符(assignment operator)。

语法拆解

部分是啥注意
name变量名(标签)不要加引号
=赋值符号两边可加空格
"小明"值字符串要加英文引号;数字不用

示例

>>> name = "小明"        ← 字符串放盒子
>>> age = 18              ← 数字放盒子(不带引号)
>>> name                  ← REPL 直接输变量名
'小明'                        ← 看见值(带引号 = 字符串)
>>> age
18                            ← 数字(不带引号)

注意:赋值这个动作没有屏幕输出——Python 默默把值放进盒子。

覆盖变量

>>> name = "小明"
>>> name = "张三"     ← 覆盖!原来的"小明"消失
>>> name
'张三'
⚠️ 常见错:name = flash(右边没引号)→ NameError。
因为 Python 把右边的 flash 当成变量名去找,但没有这个变量。
修复:加引号 → name = "flash"
第 3 章

变量名的规则

硬规矩(违反 → SyntaxError)

  • 只能用:英文字母 / 数字 / 下划线 _
  • 不能用:空格、连字符 -、标点
  • 不能用数字开头(1name ❌,name1 ✅)
  • 区分大小写(name、Name、NAME 是 3 个不同变量)
  • 不能和 Python 关键字撞名(if、for、while 等)

软建议(违反不报错但代码丑)

  • 用小写英文:name、user_age
  • 名字要有意义:user_name > x
  • 多个单词用下划线连(snake_case 风格):first_name > firstname
  • 中文变量名虽然合法,但不推荐(打字慢、不国际化)

例子对比

名字合法推荐
age✅✅
user_email✅✅
UserEmail✅⚠️ 不是 Python 风格
2nd_age❌ SyntaxError—
my-cat❌ - 当减号—
if❌ 关键字—
电影名✅ 合法⚠️ 中文不推荐
第 4 章

打印变量:REPL vs 脚本的差异

REPL 里:输变量名就能看

>>> name = "小明"
>>> name
'小明'

脚本里:必须用 print()

test.py 内容:

name = "小明"
name             ← 看似一样,实际啥也不显示

跑 python test.py:屏幕啥都没显示。

为什么?

REPL 是交互模式,Python 默认会回显你输的东西,方便即时看。

脚本是批处理模式,Python 默默执行每一行,不会主动显示任何东西——除非你明确说"打印这个"。

脚本里的正确写法

name = "小明"
print(name)         ← 必须用 print 才能看

注意 print(name) 里的 name 没引号——因为它是变量,要去盒子取值。

如果写 print("name"),会原样打印 4 个字母 name(字符串)。

第 5 章

input() 让程序问问题

核心:input() 一个函数干 3 件事

input("some")
    ↓
① 在屏幕上显示 "some"(像 print 一样)
② 暂停程序,等用户敲键盘
③ 用户回车后,把输入的内容"递回来"

这 3 件事按顺序自动完成,不需要你写 3 行代码。

语法拆解

answer = input("你叫什么名字?")
   ↑          ↑          ↑
 接住盒子   函数        提示语
部分是啥干啥
input(...)函数等用户输入,返回输入的内容
"你叫什么名字?"字符串参数屏幕上先显示,提示用户输什么
answer = ...赋值把用户输的存进 answer 变量

关于 print vs input 的"返回值"

这是个新概念,叫函数的返回值。

类比自动售货机:

  • 输入:塞钱、按按钮
  • 返回:掉一瓶水出来
水 = 售货机(钱, 按钮)         ← 接住返回的水

input 也一样:

  • 输入:括号里的提示语
  • 返回:用户键盘输入的字符串
printinput
主要工作打印到屏幕从用户那收数据
有返回值吗❌ 没有有用的返回值✅ 返回用户输入的字符串
能赋值吗x = print("hi") 没意义x = input("hi") ✅

关键认知

input() 这一行执行时,整个程序卡住等用户输入。这就是它能"问问题"的原因——程序真的会等。

第 6 章

input 返回的永远是字符串(经典坑)

核心事实

不管用户输的是字母、汉字还是数字,input() 拿回来的都是字符串。

哪怕你输 5,input() 给你的也是字符串 "5",不是数字 5。

这有啥区别?

"5" + "3"   ← 字符串 + 字符串 = "53"(粘起来)
 5  +  3    ← 数字 + 数字 = 8(算数)

实际踩坑

>>> a = input("第一个数:")     输 5
>>> b = input("第二个数:")     输 3
>>> a + b
'53'                              ← 不是 8!

这是因为 a 和 b 都是字符串。验证:

>>> type(a)
<class 'str'>

修复:用 int() 转换

>>> a = int(input("第一个数:"))    输 5
>>> b = int(input("第二个数:"))    输 3
>>> a + b
8

读法:

  1. input(...) 拿回字符串 "5"
  2. int(...) 把字符串 "5" 转换成数字 5
  3. 把数字 5 放进 a 盒子

这种"函数套函数"叫嵌套调用——内层先执行,把结果传给外层。

新工具:type() 查变量类型

type("hello")     → <class 'str'>     字符串
type(5)            → <class 'int'>     整数
type(3.14)         → <class 'float'>   小数

不知道一个变量是啥类型,用 type() 一查就明白。

为啥 input 永远返回字符串?

设计哲学:"我不知道用户想输啥,可能名字、可能数字、可能日期——保险起见全当文字处理,你需要再自己转。"

第 7 章

字符串拼接:+ 不是相加,是粘贴

同一个 +,不同行为

3 + 5            ← 数字 + 数字 = 8(数学加法)
"3" + "5"        ← 字符串 + 字符串 = "35"(粘贴)
"你好" + "世界"   ← "你好世界"

Python 看左右两边类型,决定 + 怎么干。这种"粘贴"操作专业叫拼接(concatenation)。

链式拼接

name = "小明"
print("你好," + name + "!欢迎来到编程世界")
→ 你好,小明!欢迎来到编程世界

经典坑:字符串 + 数字 → TypeError

age = 18
print("我今年" + age + "岁")     ← ❌ TypeError

Python 看到 字符串 + 数字,不知道你想拼接还是算数,直接报错。

修复:用 str() 把数字转成字符串

age = 18
print("我今年" + str(age) + "岁")    ← ✅ 我今年18岁

str() 和 int() 是反方向

函数干啥
int("5")字符串 → 数字
str(5)数字 → 字符串
type(...)查类型

实战:写一个互动问候程序

程序效果

你叫什么名字?小明                ← 你输入
你好,小明!欢迎来到编程世界 🎉    ← 程序输出

6 步流程

第 1 步:回到 PowerShell

如果在 REPL,先 exit()。

第 2 步:进练习文件夹

cd G:\研究生学习\python-练习

第 3 步:用记事本新建文件

notepad greet.py

弹出对话框点"是"新建。

第 4 步:在记事本里手打代码

name = input("你叫什么名字?")
print("你好," + name + "!欢迎来到编程世界 🎉")
⚠️ 三大守则
① 输入法切英文(Shift)
② 括号、引号、感叹号必须英文
③ 每行末尾回车换行

第 5 步:保存

Ctrl + S。标题栏从 *greet.py 变成 greet.py。

第 6 步:回 PowerShell 跑

python greet.py

逐行拆解

第 1 行 name = input("你叫什么名字?"):

  1. input 把"你叫什么名字?"打印到屏幕
  2. 暂停程序等输入
  3. 用户输入"小明"回车
  4. input 返回字符串 "小明"
  5. 赋值给 name 盒子

第 2 行 print("你好," + name + "!欢迎来到编程世界 🎉"):

  1. 拼接 "你好," + "小明" + "!欢迎来到编程世界 🎉"
  2. 结果:"你好,小明!欢迎来到编程世界 🎉"
  3. print 打印

加分挑战

name = input("你叫什么名字?")
hobby = input("你最喜欢做什么?")
print("你好," + name + "!")
print(name + "最喜欢的是" + hobby + ",听起来很棒")

🪤 真实踩坑记录

坑 1:NameError(右边没引号)
name = flash 报错——Python 把 flash 当变量名找不到。
修复:加引号 → name = "flash"
坑 2:TypeError(字符串 + 数字)
"我今年" + age(age=18)报 TypeError。
修复:"我今年" + str(age)
坑 3:SyntaxError unterminated string literal
字符串没"封口"——通常因为引号是中文全角 "。
修复:输入法切英文,所有标点重新检查。
坑 4:SyntaxError invalid character '!' (U+FF01)
用了中文全角感叹号 !。Python 不认识。
修复:换成英文 !。
坑 5:input 拿回的是字符串,不是数字
"5" + "3" = "53",不是 8。
修复:用 int() 包一层 → int(input(...))
