第 1 章

Python 到底是个啥?

一句话

Python 就是一个程序(一个 .exe 文件)——就像 QQ、微信也都是程序一样。

这个程序干嘛用?

它的工作是"翻译":

你写的代码  →  Python.exe(翻译官)  →  电脑能懂的指令  →  电脑执行
  • 你:只会写人能看懂的代码(比如 print("你好"))
  • 电脑(CPU):只会读 010101 这种二进制
  • Python.exe:站在中间,把你的代码翻译成电脑能懂的指令

所以"学 Python"的意思是:学会用 Python 这个翻译官能听懂的语言来写指令。

它在你电脑哪里?

Python 装好后通常在某个固定路径,比如 G:\study\Python\python.exe。它就是个文件——双击它会闪一下黑窗口然后消失,因为它需要你"跟它说话"才会工作。

💡 专业名词(混个脸熟):Python 这种"边翻译边执行"的程序,叫解释器(Interpreter)。跟它对应的还有"编译器(Compiler)",那是另一种翻译方式(C 语言、Java 用)。

需要精确化的一点

Python 不能听你随意说话。它能听懂的是一种有特定语法规则的"半人话半符号"语言。打个比方:

你以为可以写实际上
"请帮我打印 hello"❌ Python 看不懂
"Print hello world please"❌ Python 看不懂
print("hello world")✅ Python 懂!
🤔 思考题:用你自己的话说,Python 是什么?
看参考答案
Python 是一个翻译器,能把"用 Python 语言写的代码"翻译成电脑能执行的指令。
第 2 章

用 Python 的两种方式

方式 1:当面聊天(REPL)

你打开翻译官的门 → 进去坐下 → 说一句他翻译执行一句 → 你说"我走了" → 关门。

  • ⚡ 即时反馈——说一句立刻看到结果
  • 🚫 不留痕迹——关门走人后,刚才说的话全部忘掉
  • 🎯 适合:临时算个数、试一行小代码

专业名字叫 REPL:

  • Read(读你说的话)
  • Eval(理解 + 执行)
  • Print(打印结果)
  • Loop(循环,等下一句)

方式 2:写信给他(脚本)

你先写一封信(一个文件)把所有指令写好 → 把信交给翻译官 → 他从头读到尾全部执行 → 给你结果。

  • 📁 永久保留——这"信"是真实文件,下次还能用
  • 🔁 可重复执行——一封信可以读 100 遍
  • 🧱 可复杂——可以写几百行
  • 🎯 适合:真正的项目,所有正式程序都这样写

"信"的文件名以 .py 结尾(比如 hello.py)。

对比表

REPL脚本
像什么当面聊天写信
怎么开始打 python 进入写好 .py 文件
怎么执行一行一行说python 文件名.py
关了还在吗没了文件还在
适合试一下、玩一下真正的程序
🤔 思考题:① "赶紧算 1+1 等于几" 用哪种?② "做个每天问候我的程序" 用哪种?
看参考答案
① REPL——一次性小问题,不需要保存。
② 脚本——要重复使用、永久保留,必须存进文件。
第 3 章

第一种用法:REPL(当面聊)

怎么进?

在 PowerShell 里输:

python

回车后看到:

Python 3.14.0 (...)
Type "help", "copyright", "credits" or "license" for more information.
>>>

关键:>>> 是什么?

它是 Python 翻译官的提示符,意思是"请讲"。

提示符含义
PS C:\xxx>PowerShell 在等你输 PowerShell 命令(dir、cd)
>>>Python 翻译官在等你输 Python 代码

它们是两套语言!搞错语言会报错——比如在 >>> 后面输 dir 会出错,因为 Python 不认 PowerShell 的命令。

关于 PATH(为啥打 python 就行)

你电脑里有一份"已知程序清单"叫 PATH。Python 安装时主动加进去了,所以打短名字就能找到。

类比手机通讯录:

手机电脑
通讯录PATH 清单
人的昵称程序的短名字(python)
完整号码程序的完整路径(G:\study\Python\python.exe)

谁在 PATH 里?

  • ✅ 系统自带:notepad、calc
  • ✅ 程序员工具(装时勾选了):python、git、gh
  • ❌ 普通图形软件:wechat、qq(打了也没用)
🤔 思考题:在 PowerShell 输 wechat 回车,微信会启动吗?
看参考答案
通常不会。微信不在 PATH 清单里(它是给普通用户用的图形软件,通常双击图标启动,没必要加进 PATH)。要在终端启动微信得给完整路径。
第 4 章

print() 函数详细拆解

这一章把人生第一行 Python 代码每个符号都拆开:

print("hello world")

4 块拆分

第几块是什么干啥
① print函数的名字告诉 Python:"我要用 print 这个功能"
② (英文左括号函数的"开口"——准备接收输入
③ "hello world"字符串要打印的内容,两边必须加引号
④ )英文右括号函数的"闭口"——告诉 Python 我说完了

① print 是个 "函数"

函数 = 一个被预先做好的小工具,你给它输入,它给你输出。

  • 微波炉就是个函数:你放食物进去,按启动,它输出加热后的食物。
  • print 函数:你给它一段文字,它输出"在屏幕上显示这段文字"。

Python 内置了很多这种小工具。print 是其中最常用的一个。

② 和 ④ 那对括号 ( )

括号是函数的"嘴巴"。任何函数都长这样:

函数名(...)

括号里塞的东西叫参数(argument)——就是你"喂"给函数的输入。

⚠️ 必须是英文括号 ( ),不能是中文括号 ( )。看着像但 Python 不认,会报 SyntaxError。

③ 那对引号 " "

引号告诉 Python "里面这段是一坨文字"——这种被引号包起来的文字,Python 里叫字符串(string)。

对比:

print("hello")    ← 我要打印 "hello" 这段文字 ✅
print(hello)      ← 没引号 → Python 以为 hello 是变量名 → 报错 NameError ❌

规则:只要是给电脑看的文字,都要用引号包起来。

⚠️ 必须是英文引号 ",不能是中文引号 " "。

故意踩两种错的对照

错误 ① — NameError(名字错)

>>> print(hello)
Traceback (most recent call last):
  File "...", line 1, in <module>
    print(hello)
NameError: name 'hello' is not defined

意思:Python 看到 hello 没引号,以为是变量名,但找不到这个变量。错叫 NameError = "名字错误"。

修复:加引号 → print("hello")

错误 ② — SyntaxError(语法错)

>>> print("hi")     ← 注意:这个是中文括号
  File "...", line 1
    print("hi")
         ^
SyntaxError: invalid syntax

意思:Python 看到中文全角括号,这不在它的语法书里。错叫 SyntaxError = "语法错误"。

修复:把符号换成英文版。

读报错的核心两招

  1. 看最后一行 → 告诉你是哪种错(NameError、SyntaxError...)
  2. 看上面"出问题的那行代码" → 告诉你错在哪一行
第 5 章

怎么"关上"翻译官的门

退出 REPL

>>> exit()

拆解 exit()

  • exit 是函数的名字——和 print 一样,都是函数
  • () 是一对英文空括号

为啥括号里啥都没有?

因为 exit() 不需要参数。它的工作只有"退出",不需要你喂任何东西。

对比:

  • print("hello") = 微波炉,要放食物才能加热
  • exit() = 一个红色按钮,按一下就关机,不需要塞东西

但括号还是得有——括号是"调用函数"的标志:

exit()      ← 调用 exit 函数 ✅
exit        ← 只是提到这个名字,Python 不会"按" ❌

判断你回到 PowerShell

提示符从 >>> 变回 PS C:\xxx> = 成功退出。

第 6 章

.py 文件到底是啥?

.py 文件就是一个普通的文本文件,只是约定:里面装的是 Python 代码。

常见文件后缀

文件名后缀通常装啥
照片.jpg.jpg图片
论文.docx.docxWord 文档
电影.mp4.mp4视频
网页.html.html网页
脚本.py.pyPython 代码
笔记.txt.txt纯文本

关键认知:后缀只是"约定"

电脑不会因为你叫 .py 就自动当 Python 代码。它本质还是普通文本文件。

类比:你给猫起名叫"狗",它本质还是猫,只是名字叫狗。一个文件叫 hello.py,本质还是文本文件,只是后缀叫 .py。

那 .py 后缀有啥用?

它是给人和工具看的"标签":

  • 告诉你:"这文件里是 Python 代码"
  • 告诉编辑器:用 Python 高亮样式显示
  • 告诉自己以后:"我要拿这个给 Python 跑"
🤔 思考题:把 hello.py 改名叫 hello.banana,内容还是 print("hello")。python hello.banana 能跑吗?
看参考答案
能跑!Python 不挑后缀,只要文件内容是合法 Python 代码,什么后缀都行。但实际工作中所有人都遵守 .py 这个约定,你也得遵守,否则别人看到你的文件不知道是啥。
第 7 章

python hello.py 这一行干了啥?

2 块拆分

部分是啥作用
python命令启动 Python 翻译官
hello.py文件名告诉 Python:"请读这个文件"

关键区别:跟不跟文件名

你输后面跟啥Python 反应
python啥也没跟进入 REPL → 出现 >>>
python hello.py跟一个文件读信模式 → 读文件 → 执行 → 退出

Python 看到后面有文件名,会理解为:"哦,你不是要聊天,是给我一封信让我读完执行。我读完就走人。"

所以 python hello.py 跑完会自动退回 PowerShell,不会停在 >>>。

Python 读到 hello.py 之后做了啥

假设文件内容:

print("hello world")

Python 内心:

【接收】Python 收到 hello.py
【打开】我打开看看里面写啥
【读第 1 行】print("hello world")
   → 这是 print 函数 + 字符串 "hello world"
   → 我的工作:在屏幕上打印 "hello world"
   → 执行!屏幕显示 hello world ✅
【读第 2 行】没了,文件结束
【收工】我读完了,退出

多行脚本

.py 文件可以是很多行。比如:

print("第一行")
print("第二行")
print("第三行")

Python 会从上到下一行一行执行:

第一行
第二行
第三行

这就是脚本的威力——一次写好,每次跑都按顺序执行。

🤔 思考题:文件 hello.py 内容如下,跑 python hello.py 屏幕显示几行?哪几行?顺序? 
print("早上好")
print("中午好")
print("晚上好")
看参考答案
3 行,从上到下:早上好、中午好、晚上好。Python 严格按文件顺序执行。
第 8 章

"安全地"创建 .py 文件

问题回顾

用 echo "..." > hello.py 写 Python 文件,跑会报错:

SyntaxError: Non-UTF-8 code starting with '\xff'

原因:PowerShell 的 > 默认用 UTF-16 编码写文件,Python 只读 UTF-8。两边对不上。

解决方案 A:Set-Content(命令行)

Set-Content -Encoding utf8 -Path hello.py -Value 'print("hello")'

拆解:

  • Set-Content = 命令名("设置-内容")
  • -Encoding utf8 = 用 UTF-8 编码
  • -Path hello.py = 写到这个文件
  • -Value 'print("hello")' = 要写的内容

解决方案 B:Notepad(推荐新手用)

在 PowerShell 里:

notepad hello.py
  • 文件存在 → 打开它的内容
  • 文件不存在 → 弹对话框问"要新建吗?" → 点是 → 弹空记事本

然后直接打字写代码 → Ctrl + S 保存。记事本默认 UTF-8 编码,永远不会有编码问题。

三种方式对比

方式优点缺点适合
echo > xxx.py一行搞定❌ 编码坑不推荐
Set-Content -Encoding utf8命令行,编码正确命令长偶尔写一行小代码
notepad xxx.py✅ 可视化、所见即所得多一步打开/保存新手首选 ⭐

套路总结

notepad hello.py 和 python hello.py 是一个套路:

程序名  +  参数(文件名)
 ↓         ↓
notepad   hello.py    ← 让记事本打开这个文件
python    hello.py    ← 让 Python 跑这个文件

大部分命令都是这个结构。

第 9 章

实战:把 hello.py 跑起来

6 步流程

第 1 步:打开 PowerShell

Win + R → 输 powershell → 回车。

第 2 步:进练习文件夹

cd G:\研究生学习\python-练习

提示符变成 PS G:\研究生学习\python-练习> 表示成功。

第 3 步:用记事本打开 hello.py

notepad hello.py

第 4 步:在记事本里写代码

直接手打(不要复制粘贴,建立肌肉记忆):

print("hello world")
print("我是 undertaker-fr")
print("我会编程了")
⚠️ 写代码三大守则
① 输入法切到英文(Shift 键切换)
② 括号、引号必须英文
③ 每行末尾按回车换行

第 5 步:保存

按 Ctrl + S。标题栏从 *hello.py 变成 hello.py 表示已保存。

第 6 步:回到 PowerShell 运行

python hello.py

看到:

hello world
我是 undertaker-fr
我会编程了

🎉 你做到了!

3 行代码 → 3 行输出 → Python 按顺序执行 → 你成功写出并运行了第一个 Python 脚本。

🪤 真实踩坑记录

坑 1:用 echo > xxx.py 写 Python 文件 → SyntaxError: Non-UTF-8 code starting with '\xff'
原因:PowerShell 的 > 默认 UTF-16,Python 只读 UTF-8。
解决:用 Set-Content -Encoding utf8 ...,或更简单——notepad 文件名。
坑 2:中文输入法状态下打括号引号 → SyntaxError
print("hi")(全角) ≠ print("hi")(英文)。眼睛容易看错。
解决:写代码前按 Shift / Ctrl+Space 切到英文输入法。
坑 3:复制路径时多带了 > 字符
cd G:\xxx> 报错 "路径中具有非法字符"。
原因:从 PowerShell 提示符 PS G:\xxx> 复制时,把末尾的 > 也复制进去了。
解决:去掉末尾的 > 重输。
