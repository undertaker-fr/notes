🌱 Day 1 · 搞懂"路径"已完成

2026-04-27
🎯 目标：明白路径是什么，并亲手在终端里走进文件夹再走出来。

核心概念

路径 = 地址 电脑里每个文件/文件夹都有一个完整地址,比如 C:\Users\Administrator\Desktop
盘符 C 盘、D 盘 = 楼栋
文件夹 楼层和房间
文件 房间里的具体物品

学到的命令

命令功能例子
pwd显示当前所在位置pwd
dir列出当前文件夹内容dir
cd 文件夹进入某个文件夹（相对路径）cd Desktop
cd 完整路径用绝对路径直达任何位置cd C:\Users\Administrator
cd ..退到上一级文件夹cd ..
mkdir 名字新建文件夹mkdir my-folder

关键认知

  • 相对路径 = "去隔壁",只能在当前位置旁边时用
  • 绝对路径 = "完整地址",在哪儿都能直达
  • 提示符 PS C:\xxx> 左边的路径就是"我现在在哪"
  • Windows 不区分大小写,但 Linux 服务器严格区分,养成"原文怎么写就怎么写"的习惯

🌿 Day 2 · 文件操作已完成

2026-04-27
🎯 目标：用纯键盘创建、读取、追加、删除一个真实的文本文件。

学到的命令

命令功能例子
cls清屏cls
echo "内容"在屏幕上打印一句话echo "你好"
echo "..." > 文件把内容写进文件（覆盖）echo "笔记" > a.txt
echo "..." >> 文件追加到文件末尾（保留旧内容）echo "再加一行" >> a.txt
type 文件查看文件内容type a.txt
del 文件永久删除文件del a.txt
Remove-Item 名字删除文件或文件夹Remove-Item my-folder

实用神技巧

  1. 拖放路径：输 cd （带空格）后,把文件夹从资源管理器拖进 PowerShell,路径自动出现。
  2. Tab 自动补全：输 cd Des 按 Tab → 自动变成 cd Desktop。
  3. 粘贴：在 PowerShell 里右键 = 粘贴,或 Ctrl + V。

踩坑记录 🪤

坑 1：echo 后面忘了空格
❌ echo"内容">a.txt → 报错 "无法识别为 cmdlet"
✅ echo "内容" > a.txt
教训：命令和参数之间必须有空格,符号 > >> 两边也要加空格。
坑 2：del 是永久删除
被删的文件不会进回收站,删了就找不回来,对真文件要谨慎。
坑 3：PowerShell 看 UTF-8 文件会乱码
老版 PowerShell 默认用 GBK 字典,读 UTF-8 写的中文文件会乱码。
解决：用 Get-Content -Encoding UTF8 文件名,或者直接用 VSCode 打开。

解锁的能力

  • 能用纯键盘新建、读取、追加、删除文件
  • 理解 >（覆盖）和 >>（追加）的区别
  • 知道报错信息怎么读：先看最后一两行,关键词通常在 Exception 那行

🚀 特别任务 · 让笔记上线已完成

2026-04-27
🎯 目标：把本地的 HTML 文件部署到公网,让全世界都能访问。

核心概念

Git 给文件每次改动"拍快照",像时间机器,随时能回到任何历史版本
GitHub 程序员的"代码版朋友圈",全球 1 亿+开发者在上面存代码、晒项目
仓库（Repo） 一个项目的容器,既存文件,也存所有历史快照
部署（Deploy） 把文件放到永远开机的服务器,让别人能通过网址访问
GitHub Pages GitHub 提供的免费网页托管服务,把公开仓库变成网站
main 分支 Git 仓库的主线,默认大家把代码推到这里

学到的命令

命令功能例子
git init -b main把当前文件夹变成 Git 仓库（主分支叫 main）git init -b main
git add 文件把文件加入"待保存清单"git add index.html
git commit -m "..."把待保存清单"拍照存档"git commit -m "首次提交"
git push把本地存档"推"到 GitHub 云端git push
gh repo create用 GitHub CLI 直接建远程仓库gh repo create notes --public --push

关键认知

  • 本地 → GitHub 的核心三步：add → commit → push
  • commit message（提交信息）不是给电脑看的,是给未来的自己和队友看的——要写清楚改了啥
  • Public 仓库 = 全世界都能看你的代码;Private 仓库 = 只有你和邀请的人能看
  • GitHub Pages 网址规则：https://用户名.github.io/仓库名/
  • 第一次部署需要 1-3 分钟才能访问,后续更新只要几十秒
  • 看构建状态：gh api repos/用户名/仓库名/pages/builds/latest

解锁的能力

  • 🌐 拥有人生第一个公网网站：undertaker-fr.github.io/notes
  • 📦 拥有第一个 GitHub 仓库,GitHub 主页有了真实作品
  • 🔄 走通了"本地改 → push → 自动上线"完整工作流

🌳 Day 3 · 写下人生第一行 Python 代码已完成

2026-04-28
🎯 目标：搞懂 Python 是什么,用两种方式（REPL + 脚本）让它执行我写的代码,写出多行 print 程序并跑通。

核心概念

Python = 翻译官 一个 .exe 程序,把人写的 Python 代码翻译成电脑能执行的指令
解释器（Interpreter） Python 这种"边翻译边执行"的程序的专业名字
REPL（即时模式） 当面聊天:输一行执行一行,关了就忘
脚本（.py 文件） 写信:把代码存进文件,Python 读完执行
函数（function） 预先做好的小工具,给输入得输出。print() exit() 都是
参数（argument） 塞进函数括号里的"输入",比如 print("hi") 里的 "hi"
字符串（string） 被一对英文引号包起来的文字,Python 把它当"一坨文本"处理
PATH 环境变量 电脑的"程序通讯录":在里面的程序能打短名字调用,不在的要打完整路径
提示符的两种形态 PS C:\xxx> = PowerShell 在等;>>> = Python 在等
NameError vs SyntaxError 前者"名字错"(常因忘加引号),后者"语法错"(常因中文标点)

学到的命令

命令功能例子
python启动 Python 进入 REPL（交互模式）,提示符变 >>>python
python 文件.py让 Python 读这个文件并执行,跑完自动退出python hello.py
print("...")Python 函数,把括号里的字符串打印到屏幕print("hello")
exit()Python 函数,退出 REPL 回到 PowerShell（空括号也得有）exit()
notepad 文件用记事本打开/创建文件,默认 UTF-8 编码,推荐用来写 .pynotepad hello.py

关键认知

  • 跟 Python 沟通有两种方式:REPL（试一下）和脚本（真程序）。后者占工作 99%
  • 函数的格式永远是 函数名(参数),括号是"调用"的标志,空括号也得有
  • 字符串必须用英文引号包起来。中文引号 " 看着像但 Python 不认
  • .py 后缀只是约定,本质是普通文本文件。改成 .banana Python 也能跑
  • 读报错先看最后一行:告诉你是哪种错(NameError / SyntaxError ...)
  • 写代码前先确认输入法是英文(En)。括号引号必须英文,这是肌肉记忆

踩坑记录 🪤

坑 4:用 echo > xxx.py 写 Python 文件 → SyntaxError: Non-UTF-8 code starting with '\xff'
原因:PowerShell 的 > 默认用 UTF-16 编码写文件,Python 只读 UTF-8。
解决:用 Set-Content -Encoding utf8 ...,或者更简单——直接用 notepad 文件名 写代码。
坑 5:中文输入法状态下打括号引号 → SyntaxError
print（"hi"）(全角) ≠ print("hi")(英文)。眼睛容易看错。
解决:写代码前按 Shift / Ctrl+Space 切到英文输入法,再开始打。

解锁的能力

  • 🐍 写下并执行了人生第一行 Python 代码 print("hello world")
  • 📝 能用记事本创建并编辑 .py 文件,避开编码坑
  • 🎬 能让 Python 按顺序执行多行代码（脚本模式）
  • 🐛 能识别两种基础报错:NameError(名字错) 和 SyntaxError(语法错)
  • 🚪 能自由进出 REPL:python 进 → exit() 出
